## ما الجديد في v7.1 — إصلاحات جيوديسية حرجة

### 🔴 العيب الأول: الاتجاهات كانت خاطئة تماماً

الكود القديم في `calcSides()`:

```javascript
const brg = turf.bearing(c, mid);   // ❌ من مركز القطعة إلى منتصف الضلع
```

هذا يحسب **موقع الضلع بالنسبة للمركز**، وليس **اتجاه الضلع**. النتيجة:
ضلع يمتد شرقاً يُوصف بأنه "جنوبي غربي" لمجرد أنه يقع جنوب غرب المركز.

الإصلاح:

```javascript
const az = BannaaGeoFix.bearing(a.lng, a.lat, b.lng, b.lat);  // ✅ على طول الضلع
```

الآن كل ضلع له:
- `dir` — اتجاه الضلع نفسه
- `face` — اتجاه الواجهة (عمودي على الضلع نحو الخارج) ← هذا ما يهم المعماري
- `az` + `dms` — الأزيموث بالدرجات وبصيغة المساحين

### 🟠 العيب الثاني: المساحة منحازة +0.27%

`turf.area` يستخدم كرة بنصف قطر متوسط (6,371,008.8 م). عند خط عرض العراق
(33.4°) هذا يعطي انحيازاً ثابتاً. استُبدل بحساب جيوديسي على إهليلج WGS84
بنصف القطر الجاوسي المحلي.

| | القيمة |
|---|---|
| قبل | 1560.4 م² |
| بعد | 1556.26 م² |
| المرجع (pyproj/PROJ) | 1556.26 م² ✅ |

### 🟠 العيب الثالث: ادعاء دقة غير موجودة

كان يعرض `1560.4` — خانة عشرية توحي بدقة ±10 سم².
الحقيقة بدقة GPS هاتف (±5 م): **1556 ± 181 م² (12%)**.

الآن يُحسب ويُعرض عدم اليقين مع درجة ثقة ملوّنة.

### ✨ إضافة: التقاط بمتوسط مرجّح (زر 📌)

يجمع قراءات لمدة 20 ثانية ويأخذ المتوسط المرجّح بـ 1/دقة².
يقلل الخطأ **3–5 أضعاف** مقارنة بقراءة واحدة.

## البنية

```
index.html            التطبيق (Leaflet + Turf + الواجهة)
bannaa-geo-patch.js   النواة الجيوديسية المصححة
vercel.json           إعدادات النشر
```

`bannaa-geo-patch.js` يُحمَّل **بعد** Turf ويرقّعه تلقائياً — فأي كود قديم
يستدعي `turf.area()` يحصل على النتيجة الصحيحة دون تعديل.

## النشر

مربوط بـ Vercel — أي `commit` على `main` ينشر تلقائياً.

## الـ Worker

الذكاء الاصطناعي والدفع يمران عبر Cloudflare Worker منفصل
(`bannaa-worker-template.js`). حدّث `S.proxy` في التطبيق ليشير إليه.

المتغيرات المطلوبة في Cloudflare:
- `ANTHROPIC_API_KEY`
- `ALLOWED_ORIGIN` (نطاق Vercel بعد النشر)
